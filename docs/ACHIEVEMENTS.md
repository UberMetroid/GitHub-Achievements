This guide covers all GitHub achievements and how to earn them legitimately.

## Detailed Tactics

### Pull Shark (Merged PRs)

**Thresholds:** 2 → 16 → 128 → 1024 merged PRs

**How to earn:**
1. Find repos with "good first issue" labels
2. Read the contribution guidelines
3. Make small, focused changes
4. Respond promptly to review feedback
5. Be patient - some repos take weeks to review

**Tips:**
- Start with your own repos for quick wins
- Look for docs-only contributions first
- Fork popular libraries and fix typos/bugs

### Galaxy Brain (Accepted Answers)

**Thresholds:** 2 → 8 → 16 → 32 accepted answers

**How to earn:**
1. Enable Discussions on your repos
2. Browse other repos' Q&A sections
3. Answer questions with working solutions
4. Include code examples when possible
5. Follow up to see if your answer helped

**Tips:**
- Focus on popular repos with many questions
- Answer questions in your areas of expertise
- Be first with a correct answer

### Starstruck (Repository Stars)

**Thresholds:** 16 → 128 → 512 → 4096 stars

**How to earn:**
1. Build something useful others need
2. Write clear documentation
3. Add a good README with examples
4. Promote on Reddit, Twitter, Hacker News
5. Submit to relevant curated lists

**Tips:**
- Solved problems gain more stars than exercises
- Good first projects: CLI tools, APIs, libraries
- Star your own repos to get started

### Pair Extraordinaire (Co-authored PRs)

**Thresholds:** 1 → 10 → 24 → 48 co-authored PRs

**How to earn:**
1. Find a collaborator
2. Pair on a feature or bug fix
3. Use co-author commits: `git commit -m "Fix" -m "Co-authored-by: name <email>"`
4. Both must be collaborators on the PR

**Tips:**
- Great for mentorship pairs
- Both people get credit
- Works with any co-author, not just maintainers

### Quickdraw (Fast Close)

**One-time achievement**

**How to earn:**
1. Create an issue describing a simple bug
2. Fix it immediately in a commit
3. Close the issue within 5 minutes

**Tips:**
- Easiest in your own repos
- Small typo fixes work well
- Requires opening issue + fix in same session

### YOLO (No Review Merge)

**One-time achievement**

**How to earn:**
1. Create a PR in your own repo
2. Have repo settings allow merging without review
3. Merge the PR without approval

**Tips:**
- Only for repos where you have admin access
- Not recommended for production code
- Consider enabling required reviews afterward

### Public Sponsor

**One-time achievement**

**How to earn:**
1. Go to a project's GitHub Sponsors page
2. Select a sponsorship tier
3. Complete payment

**Tips:**
- Sponsors help maintainers afford to work open source
- Even $1/month counts
- Your sponsor badge shows on your profile

### Hacker (First Public Repo)

**One-time achievement**

**How to earn:**
1. Create any public repository
2. Add at least one file (README counts)

**Tips:**
- Easiest achievement - just create a repo
- Name it something meaningful
- Add a README explaining the project

### Founder (First Repo)

**One-time achievement**

**How to earn:**
1. Create your first repository (public or private)

**Note:** This was previously awarded automatically. Now it's earned when you create your first repo.

### Developer (Profile Picture)

**One-time achievement**

**How to earn:**
1. Go to GitHub settings
2. Upload a profile picture

**Tips:**
- Use a recognizable photo or icon
- Shows on all your contributions

### Llama (1000 Contributions)

**One-time achievement**

**How to earn:**
1. Make 1,000 contributions in a single year
2. Contributions include: commits, PRs, issues, reviews

**Tips:**
- Make small contributions consistently
- Fix one typo daily = 365 contributions/year
- Use GitHub's contribution graph to track

### Arctic Code Vault

**One-time achievement (2020 only)**

**How to earn:**
- This achievement was awarded in 2020 for any open source contribution
- Cannot be earned anymore

## Guardrails

- No fake PRs or spam
- No buying stars or reviews
- No manipulating engagement
- All achievements should be earned through genuine contributions

## Why Legit Achievements Matter

1. **Professional credibility** - Real achievements show actual skills
2. **Learning** - The work that earns achievements teaches you real skills
3. **Community** - Genuine contributions help others
4. **No risk** - Fake achievements can lead to account suspension
